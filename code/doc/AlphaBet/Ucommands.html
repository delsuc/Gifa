UNIT 

UNIT value
	The context UNIT will determine the unit that will be used
by the program for displaying dataset cordinates  
UNIT can take the values INDEX (number of 
channel) HERTZ (based on SPECW and OFFSET), PPM (HERTZ divided by 
FREQ), SECOND (usefull only for FID), TABULATED (values are taken from 
the TAB buffer) or DAMPING (in damping values). All other values are 
invalid.

related contexts : $UNIT


 see also : AXIS calib choose_unit DMAX DMIN FREQ OFFSET pklist PLOTAXIS point PUT SPECW UNIT_Y

UNIT_Y 

UNIT_Y value
        The context UNIT_Y will determine the unit used for the vertical
axis on 2D display
UNIT_Y can take the values INDEX (number of
channel) HERTZ (based on SPECW and OFFSET), PPM (HERTZ divided by
FREQ), SECOND (usefull only for FID), TABULATED (values are taken from
the TAB buffer) or DAMPING (in damping values). All other values are
invalid.

UNIT_Y is reset to UNIT wheneever the UNIT command is used.

related contexts : $UNITY choose_unit


 see also : UNIT

UNREF 

When this command is issued, the command line will not refresh the 
current display. Typically used at the end of a command line in macro :

read file  unref	; read the file but does not display it

This is the inverse of REF


 see also : CLEAR REF UNREF

upzoom

located in /usr/local/gifa/macro Show macro 
 upzoom permits to propagate a zoom window to a higher dimension

Normally, each dim buffer (i.e. 1D, 2D 3D) has an independent zoom definition.
 however, commands like
    ROW, COL, PROJ (2D->1D)
    PLANE, PROJ (3D->2D)
    VERT (3D->1D)   etc...
 copy the zoom definition from the higher dimension to the lower one.

 upzoom realizes exactly the opposite.
 if you are in nD, upzoom will copy the current zoom definition to (n+1)D
  using the last commands to guess the axis.

 
 see also : ZOOM zoom3di

user_apod

located in /usr/local/gifa/macro Show macro 
 user_apod func($i)

 func($i) is entered as a string which defines the function to be applied
 as an apodisation function.
 $i will run from 1 to the size of the data_set
 no blank should be inserted in the definition of func($i)

 all context can be used
 $pi is predefined

 
 see also : user_apod_2d

USWA 

USWA {Fx}
Take a complex data set in internal representation ( i.e. odd points 
real and even points imaginary), and put the real part on the left 
side, and the imaginary part on the right side of the data set. This 
is the opposite of SWA.
 
related contexts : $ITYPE_1D $ITYPE_2D $ITYPE_3D


 see also : FLIP FLOP REAL SWA
